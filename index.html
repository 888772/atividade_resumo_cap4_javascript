Tornando Páginas Web Interativas com JavaScript - RESUMO

O capítulo apresenta os fundamentos essenciais do JavaScript, a linguagem responsável por adicionar comportamento dinâmico e interatividade às páginas web. São abordados conceitos como variáveis, constantes, tipos de dados, operadores, estruturas de controle, funções, tratamento de erros, objetos e arrays.

Os principais tópicos discutidos incluem:

  • Inclusão do JavaScript: Formas de integrar código JavaScript em páginas HTML, incluindo arquivos externos (.js) e código direto no HTML.
  • Variáveis e Constantes: Armazenamento de dados com let (variáveis) e const (constantes), além dos tipos de dados disponíveis.
  • Operadores: Operadores aritméticos, lógicos, de comparação, de concatenação e outros disponíveis na linguagem.
  • Estruturas de Controle: Controle do fluxo do programa por meio de condicionais (if/else, switch) e laços de repetição (for, while, do-while).
  • Funções: Blocos de código reutilizáveis, definidos via declaração, expressão ou arrow functions, com parâmetros e valores de retorno.
  • Objetos e Arrays: Agrupamento de dados e listas de valores para organizar informações de forma eficiente.

4.1 Introdução

4.1.1 Incluindo JavaScript

O código JavaScript pode ser incluído em uma página web de diferentes formas. A abordagem recomendada é armazená-lo em um arquivo separado com extensão .js e referenciá-lo no HTML com a tag <script> e o atributo src.

1º Arquivo JavaScript Externo (recomendado)

<script src="scripts/main.js"></script>

2º JavaScript Direto no HTML

<script>
  alert('Hello World');
</script>

Também é possível usar o elemento <noscript> para exibir conteúdo alternativo quando o JavaScript está desabilitado no navegador.

4.1.2 Exibindo Caixas de Diálogo

O JavaScript oferece três funções padrão para caixas de diálogo:

  • alert() — exibe uma caixa de dica simples
  • confirm() — exibe uma caixa de confirmação com dois botões (OK e Cancelar)
  • prompt() — abre uma caixa de entrada de texto

Na prática, essas caixas são pouco usadas pois têm funcionalidade limitada e aparência não customizável.

4.1.3 Usando o Console de Desenvolvimento

O console de desenvolvimento é uma ferramenta essencial para testar e depurar código JavaScript. Para escrever no console, usa-se o objeto console:

Método Descrição
console.log() Imprime uma mensagem normal no console
console.debug() Imprime uma mensagem de depuração
console.error() Exibe uma mensagem de erro
console.info() Exibe uma mensagem informativa
console.warn() Emite um aviso no console
console.clear() Limpa todas as mensagens do console

4.2 Variáveis, Constantes, Tipos de Dados e Operadores

4.2.1 Definindo Variáveis

Variáveis são criadas com a palavra-chave let. Podem ser declaradas e inicializadas em etapas separadas ou juntas:

let firstName;                // declaração
firstName = 'João';        // inicialização
let lastName = 'Silva';  // declaração e inicialização juntas

4.2.2 Definindo Constantes

Constantes são declaradas com const. Seu valor é atribuído uma única vez e não pode ser alterado. Por convenção, são escritas em letras maiúsculas:

const MAXIMO = 5000;

4.2.3 Usando Tipos de Dados

O JavaScript possui seis tipos de dados principais:

  • number — números inteiros e decimais: const n = 42; ou const d = 3.14;
  • string — sequências de caracteres: const nome = 'Maria'; ou const nome = "Maria";
  • boolean — valores lógicos: const ativo = true; ou const admin = false;
  • null — ausência intencional de valor
  • undefined — variável declarada mas sem valor atribuído
  • object — agrupamento de propriedades e métodos

Também existem template strings (com crase `), que permitem usar ${variavel} como placeholder diretamente no texto.

4.2.4 Usando Operadores

Os principais tipos de operadores disponíveis no JavaScript são:

  • Aritméticos: +, -, *, /, %, **, ++, --
  • Atribuição: =
  • Concatenação: + (para juntar strings)
  • Lógicos: && (AND), || (OR), ! (negação)
  • Bit a bit: &, |, ^, ~, <<, >>
  • Comparação: <, >, <=, >=, ==, ===, !=, !==

4.3 Usando Estruturas de Controle

4.3.1 Condicionais e Ramificações

A instrução if executa um bloco de código somente se uma condição for verdadeira. Com else if e else, é possível criar múltiplos caminhos:

if (senhaCurta) {
  mensagem = 'Senha deve ter ao menos 10 caracteres.';
} else if (senhaLonga) {
  mensagem = 'Senha não pode ter mais de 50 caracteres.';
} else {
  mensagem = 'Senha válida!';
}

O switch permite testar uma variável contra vários valores possíveis, executando o bloco correspondente ao valor encontrado. O break encerra a execução do switch após cada caso.

4.3.2 Usando Laços de Repetição

Existem três tipos de laços em JavaScript:

  • for (laço de contagem): repete instruções com base em um contador. Possui inicialização, condição e expressão de incremento.
  • while (laço controlado pelo início): verifica a condição antes de cada iteração. Pode não executar nenhuma vez.
  • do-while (laço controlado pelo fim): verifica a condição após cada iteração. Executa ao menos uma vez.

for (let i = 1; i <= 10; i++) { console.log(i); }

4.4 Funções e Tratamento de Erros

4.4.1 Definindo e Chamando Funções

Funções agrupam código em blocos reutilizáveis. Existem três formas principais de defini-las:

  • Declaração de função: function nomeDaFuncao() { ... }
  • Expressão de função: const fn = function() { ... }
  • Arrow function: const fn = () => { ... }

Para chamar uma função, basta escrever seu nome seguido de parênteses: nomeDaFuncao();

4.4.2 Parâmetros e Valores de Retorno

Funções podem receber valores por meio de parâmetros e retornar resultados com return:

function soma(x, y) {
  let resultado = x + y;
  return resultado;
}
const z = soma(5, 6); // z = 11

4.4.3 Tratamento de Erros

Três tipos de erros podem ocorrer em JavaScript:

  • Erros de sintaxe: violação das regras sintáticas da linguagem (ex.: palavra-chave errada)
  • Erros de execução: ocorrem durante a execução, como tentar acessar uma variável inexistente
  • Erros lógicos (bugs): causados por lógica incorreta no programa

Para tratar erros de execução, usa-se o bloco try-catch: o código que pode lançar erros fica no bloco try, e a resposta ao erro fica no bloco catch. Erros são lançados com throw new Error('mensagem').

4.5 Objetos e Arrays

4.5.1 Usando Objetos

Objetos agrupam dados relacionados em propriedades e métodos. A forma mais simples de criá-los é a notação literal (object-literal notation), com chaves {}. O acesso às propriedades e métodos é feito com notação de ponto:

const livro = {
  titulo: 'JavaScript – O Guia Completo',
  preco: 54.99,
  autor: 'Philip Ackermann',
  imprimirDescricao() {
    console.log(`${this.autor}: ${this.titulo}`);
  }
}
console.log(livro.titulo); // "JavaScript – O Guia Completo"
livro.imprimirDescricao(); // "Philip Ackermann: JavaScript – O Guia Completo"

4.5.2 Usando Arrays

Arrays são listas de valores, criados com colchetes []. O índice começa em 0. Novos elementos são adicionados com push(). É comum iterar arrays com laços for.

Método Descrição
push() Insere um elemento no final do array
pop() Remove o último elemento do array
shift() Remove o primeiro elemento do array
forEach() Aplica uma função a cada elemento
filter() Filtra elementos com base em um critério
map() Transforma os elementos do array
sort() Ordena os elementos do array
join() Converte o array em uma string
slice() Extrai partes do array
splice() Adiciona ou remove elementos em qualquer posição

    4.6 Resumo e Perspectivas

    4.6.1 Pontos-chave

    • JavaScript pode ser integrado de várias formas, mas o ideal é manter o código em arquivos separados .js.
    • Variáveis são declaradas com let e constantes com const.
    • Os tipos de dados incluem: number, string, boolean, null, undefined e object.
    • Vários tipos de operadores estão disponíveis: aritméticos, lógicos, de comparação, de concatenação e bit a bit.
    • O fluxo de controle é gerido por condicionais (if/else, switch) e laços (for, while, do-while).
    • Funções definem partes reutilizáveis do programa, podendo receber argumentos e retornar valores.
    • Erros de execução são tratados com blocos try-catch.
    • Objetos agrupam dados e funcionalidades; Arrays funcionam como listas de valores.
